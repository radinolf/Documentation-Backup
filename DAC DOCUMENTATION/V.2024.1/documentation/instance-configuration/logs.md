### Introduction

Logs are a fundamental tool for troubleshooting issues and bugs of both the application you are building or App Composer itself.

When you contact the Decisyon customer support, most probably you will be asked to provide logs.

App Composer allows to download logs of both Design Studio and Runtime.

### &#x20;Log in Runtime

The DAC generates logs that are addressed to the standard output or stored in a file to be later downloaded.&#x20;

Therefore, it is feasible to implement and use an external logs aggregator services configured to work on the Kubernetes cluster to centralize the management of logs. This approach is invaluable for debugging purposes or for real-time monitoring, allowing developers or system administrators to view logs directly on the console or at a terminal while the application is running.&#x20;

Some examples of popular logs aggregators include solutions such as New Relic, Datadog and ELK.

Itâ€™s also possible to retrieve logs from the [kubectl](https://kubernetes.io/docs/tasks/tools/) command line:

```

### Log in Design Studio

From Design Studio, go to the Help menu and select Logs. It will open the folders in your operating system where the logs of Design Studio are saved.

<figure><img src="../../.gitbook/assets/image (373).png" alt=""><figcaption></figcaption></figure>

### Log rotation

The log files are stored in a **.zip** file. Each file reaches a maximum size of of 20 megabytes and is automatically deleted a week after its creation.&#x20;

If several logs are produced in the same day, then the file name will have a date and an increasing progressive number.&#x20;

This is applicable for both the logs of Design Studio and Runtime.

```
2023-10-31 10:20:12,792 INFO com.decis.sessi.manager [http-nio-8080-exec-602] [SessionManager] New Session Created C7B27CBBA42F4C03340187618B1C47C4code

* **Timestamp**: 2023-10-31 10:20:12,792
  * This indicates that the event was logged on October 31, 2023, at 10:20:12 with millisecond precision (792 milliseconds) in UTC time zone.
* **Log Level**: INFO
  * This indicates that the event has an "INFO" log level, which typically means it's an informational message and not critical.
* **Log Origin**: com.decis.sessi.manager
  * This is the log's origin and indicates that the event was generated by a class or component named "`com.decis.sessi.manager`."
* **Thread**: \[http-nio-8080-exec-602]
  * This part of the log indicates the name of the thread that generated the event, which in this case is "`[http-nio-8080-exec-602]`."
* **Message**: \[SessionManager] New Session Created C7B27CBBA42F4C03340187618B1C47C4
  * This is the main log message. It indicates that the "SessionManager" has successfully created a new session and provides a unique identifier for this new session, which is "C7B27CBBA42F4C03340187618B1C47C4."

In summary, this log line reports that the "SessionManager" has successfully created a new session with a unique identifier "C7B27CBBA42F4C03340187618B1C47C4" on October 31, 2023, at 10:20:12 with millisecond precision. The log level is "INFO," indicating it's an informational message.
