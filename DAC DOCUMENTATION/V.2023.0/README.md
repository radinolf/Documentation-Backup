# Welcome

Thanks to **App Composer,** users can create custom applications for data analysis and management, providing a graphical and intuitive visualization. The documentation provides detailed information on how to configure the application and use technical properties to customize your application.

* The user manual [**documentation** ](./#documentation)for the DAC is divided into three sections to help users easily find the information they need and use the software efficiently and effectively.The first section is the general documentation, which provides an overview of the software and its features. This section is useful for users who are using the software for the first time and want to understand its basic functionality.
* The second section is the [**Release Notes**](./#release-notes), which describe the new features and bug fixes introduced in each new version of the software. These notes are useful for users who want to know what has been updated and improved in the latest version of the software.
* Finally, there is the [**Installation Guide**](./#installation-guides), which provides detailed instructions for installing and configuring the software. This section is important for users who are installing the software for the first time and want a step-by-step guide to ensure that the installation is done correctly.

## Documentation

In the **Documentatiom**, you will find instructions on how to set up security fiter to protect application data, how to associate data from various sources such as databases, files, or APIs, and how to map them in your application.

In addition, instructions will be provided on how to create data analysis **Pages** using graphical widgets available in DAC. With **widgets**, you can display data clearly and intuitively, choosing from various graphical display options. Also, DAC allows exporting analysis results for sharing with other team members.

In summary, DAC documentation provides all the necessary information to use App Composer and create custom applications for data analysis and management. Detailed instructions and practical examples help users understand the software's features and use them effectively to improve their data analysis and management activities. Whether you're just starting or an experienced veteran developer, there's something for everyone.

Whether you are a beginner in the development world or an experienced developer, the DAC software offers a wide range of features to meet your needs.

If you are interested in using custom widgets in your project, you can find all the information you need in the DAC Widget Hub. Here, you will find complete documentation for each supported widget, with detailed instructions on how to use each widget and how to integrate it into your project.

Regardless of your level of experience, the DAC Widget Hub documentation is a valuable resource to help you use custom widgets in your project effectively and smoothly. Thanks to the complete documentation and detailed instructions, you will be able to use all the widgets supported by DAC software to improve the functionality and visualization of your projects.

## Release Notes

The Release Notes provide important information for developers who are working with the low-code software. By reviewing the What's New, Change Log, and API Change Log sections, developers can stay up-to-date on the latest features and changes, as well as any potential impacts on their existing applications.

* [**What's New**](./#whats-new)**:** This section provides an overview of the new features and improvements that have been added to the software. It may include new widgets, new data sources, new capabilities, and any other new functionality that has been added to the software.
* [**Change Log:**](./#change-log) The Change Log section describes any changes that have been made to the software. This can include bug fixes, performance improvements, and other changes that have been made to the software since the previous release. It is important to review this section carefully to understand how the changes may impact your existing applications.
* [**API Change Log:**](./#api-change-log) This section documents any changes that have been made to the APIs (Application Programming Interfaces) that the software uses. APIs are used to allow different software systems to communicate with each other, and any changes to the APIs can have a significant impact on existing applications. This section will describe any changes to the APIs and provide guidance on how to update your applications to use the new APIs.
